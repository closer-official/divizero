# SNS投稿ルール設計図 v1.0
## 小林薫之介 / Divizero｜X・Instagram共通

---

## 0. この設計図の使い方

このファイルをAIに渡して以下を入力するだけで、毎回一定品質の投稿が生成される。

```
【入力フォーマット】
投稿タイプ：（A〜Eから選択）
テーマ：（例：「LPがない人が損していること」）
実体験・ネタ元：（あれば。なければ「なし」）
媒体：X / Instagram / 両方
```

## 1. アカウントの立ち位置（全投稿の前提）

**売っているもの：** LP・HP制作の「相談窓口」
**ターゲット：** X・インスタで発信している個人事業主・コンサル・アフィリエイター・無形商材セラー
**ターゲットは2層：**
- A層：LPが必要と分かっていて頼み先を探している
- B層：LPの必要性にまだ気づいていない（こちらの方が多い）

**トーンの核心：**
「教える人」ではなく「同じ側にいる人」として書く。
発信者の視点・悩みを持つ人間として共感から入り、気づきを渡す。
コンサル臭・分析臭・上から目線は即アウト。

## 2. 投稿タイプ定義（A〜E）

### タイプA｜気づき投稿（最頻出・週3〜4本）

**目的：** B層に「そうか、ページが要るんだ」と気づかせる
**構造：**
```
① 刺さる一行（問いかけ or 逆説）
② 具体的な場面描写（「こういう人いる」）
③ 気づき or 視点の転換
④ 締め（結論 or 静かな問いかけ）
⑤ ※事業開示は入れない or 最後に1行だけ
```

**文字数：** X=140〜250文字 / インスタキャプション=300〜500文字
**禁止：** 「私のサービスが解決します」で終わる / 「ぜひDMを」で終わる / 教訓で終わる
**良い終わり方の例：**
- 「あなたのページは、その役割を果たせていますか。」
- 「…という話でした。」（あえて締めない）
- 「これ、意外と見落とされてますよね。」

**例文（テーマ：熱量が冷める話）：**
```
LINEに登録してもらった瞬間が、いちばん熱量高い。

でも大体の人は、その後に自動メッセージが届くまで
数時間〜数日、何もない。

熱は、その間に冷える。

登録直後に「あなたのことを待っていました」という
コンテンツが見える状態になっているか。

ページ1枚で、その時間を埋められる。
```

---

### タイプB｜ストーリー投稿（週1〜2本）

**目的：** 人間を見せる。信頼と親しみの貯金を作る
**構造：**
```
① 出来事 or 気づきの瞬間（具体的に）
② そこから何を考えたか
③ 今の自分の立場との接続（ただし説教にしない）
```

**文字数：** X=200〜300文字 / インスタ=400〜700文字
**禁止：** 成功話・実績自慢・「おかげさまで」系
**推奨テーマ：**
- なぜこの仕事を始めたか（繰り返し使って良い・角度を変える）
- 失敗した話
- 「これは違うな」と思った瞬間
- 誰かとの会話から気づいたこと

**例文（テーマ：始めた理由）：**
```
正直に言うと、最初は自分のためだった。

発信してるのに問い合わせが来ない時期が続いて、
「何が足りないんだろう」とずっと考えてた。

たどり着いたのは、受け皿の話だった。

発信は悪くない。ページがなかった。
あるいは、あっても機能していなかった。

それから人の相談に乗るようになったら、
同じところで詰まってる人が多くて。

だから窓口を作った。
自分が欲しかったものを、渡せる人間になりたくて。
```

---

### タイプC｜ノウハウ投稿（週1本）

**目的：** A層への即時信頼獲得。「この人は分かってる」を作る
**構造：**
```
① 結論から始める（ぼかさない）
② 理由 or 仕組みの説明（2〜3点）
③ 具体例
④ 読者への問いかけ（チェックリスト形式でも可）
```

**文字数：** X=250〜350文字 / インスタ=500〜800文字
**禁止：** 専門用語の羅列 / 「〜が大切です」で終わる / 網羅しようとする
**推奨テーマ：**
- LPで最初に書くべきこと
- LINE登録ページで外してはいけない1点
- 価格の見せ方で変わること
- スマホで見た時に離脱される理由
- ファーストビューの役割

**例文（テーマ：ファーストビュー）：**
```
LPで最初に決まることは、「続きを読むか」だけ。

ファーストビューの仕事は売ることじゃない。
「自分のための話だ」と思わせること。

だから最初の1文は問いかけが強い。

「月収を増やしたいですか？」より
「発信しているのに、なぜか売れない人へ。」の方が
刺さる人だけが残る。

あなたのLPの最初の1行、
今すぐ見返してみてください。
```

---

### タイプD｜共感投稿（週1本）

**目的：** B層に「そうそう、それ」と思わせてフォローさせる
**構造：**
```
① あるあるの場面を描写（断定・共感を引く）
② 「でも実は〜」で視点転換
③ 静かに終わる（解決策を売らない）
```

**文字数：** X=100〜200文字（短い方が刺さる） / インスタ=200〜400文字
**禁止：** 解決策をページ末尾に売り込む / 「こういう時はDMを」
**推奨テーマ：**
- 「発信してるのに反応がない」
- 「フォロワーはいるのに売れない」
- 「LP作ったのに問い合わせが来ない」
- 「制作会社に断られた/高すぎた」
- 「何から始めればいいか分からない」

**例文（テーマ：発信してるのに売れない）：**
```
発信してるのに売れない、は
発信が悪いとは限らない。

受け皿が弱いだけのことが、多い。

発信は集客。
ページは説得。

この2つは、別の仕事。
```

---

### タイプE｜事業開示投稿（月1〜2本）

**目的：** 自分が何者かを自然に示す。売り込みにしない
**構造：**
```
① 第三者視点 or 相談を受けた話から入る
② 「私はこういう仕事をしています」を自然に開示
③ どんな人の相談を受けているか
④ CTAは「DM or LINE」の1行のみ。プッシュしない
```

**文字数：** X=150〜250文字 / インスタ=300〜500文字
**禁止：** 「今なら特別価格」「期間限定」「残り◯枠」「ぜひお気軽に」
**推奨タイミング：** A・B・Cの良い投稿の直後（信頼残高が高い時）

**例文：**
```
「LPが欲しいんですけど、何から始めればいいか分からなくて」

こういう相談、最近よく来る。

私はLP・HP制作の相談窓口をやっていて、
発信者・コンサル・アフィリの人からの依頼が多い。

まず無料でデモを見てもらって、
気に入ったら進む、という流れ。

気になる方はDMでどうぞ。
```

## 3. 投稿禁止事項（全タイプ共通）

以下が1つでもあれば廃棄して再生成。

**内容面**
- 「〜が大切です」で終わる（教訓臭）
- 「ぜひDMを」「お気軽に」「お待ちしています」（押し売り臭）
- 「おかげさまで」「ご好評をいただき」（慇懃無礼）
- 競合批判・比較（「他の制作会社と違い〜」）
- 根拠のない断言（「必ず伸びます」「絶対に売れます」）
- マーケ用語の乱用（導線設計・ファネル・コンバージョン・訴求）

**構造面**
- 最初の1行が「私は〜」で始まる（自分語りスタート）
- 3行以上連続した体言止め（読みにくい）
- 箇条書きが5点以上（まとめ感・業者感）
- 最後がCTAで終わる（タイプE以外）

## 4. ビジュアル生成ルール（投稿画像）

### X：画像は基本つけない or テキスト画像1枚

Xはテキスト主体。画像を毎回つけると「業者の量産コンテンツ」に見える。
**つける場合の条件：** キャプションの補足が視覚的に必要な時のみ（図解・比較表）

**テキスト画像のデザイン指定（ChatGPTへの指示）：**
```
1200×675px
背景：#FAFAF8（オフホワイト）
左に縦のアクセントライン：#3D6B4F（深い緑）、幅4px
テキスト：#1A1A1A、フォントはゴシック系
メインコピー：フォントサイズ大（48〜64px相当）
サブコピー：フォントサイズ小（24〜28px相当）
右下に「@21closer_ai」のウォーターマーク（小・薄め）
余白多め・シンプル
```

---

### Instagram：画像は毎回つける

インスタは画像が命。ただし「映え狙い」は不要。**読まれる画像**を作る。

**画像タイプ別指定：**

#### ① テキストメイン画像（タイプA・D投稿に）
```
1080×1080px（正方形）
背景：#FAFAF8 or #FFFFFF
大きなテキスト1行〜2行（投稿の核心となる一文）
例：「熱は、冷める前に受け止める。」
フォント：ゴシック系・太め
アクセント：左端or下端に#3D6B4Fのライン
右下：@tadanosuke.divizero（小・薄め）
```

#### ② 図解・比較画像（タイプC投稿に）
```
1080×1080px
Before/After形式 or 左右比較
左：課題の状態（薄いグレー背景）
右：改善後の状態（アクセント色背景）
テキストは最小限・アイコンで補足
```

#### ③ 人物画像（タイプB・E投稿に）
```
プロフィール写真（確定済みスーツ写真）を使用
テキストオーバーレイで一言を添える
例：写真の上部に「なぜ、この窓口を作ったか。」
半透明の暗めオーバーレイ + 白テキスト
```

**ChatGPTへの標準指示文：**
```
Instagram投稿用画像を作ってください。
サイズ：1080×1080px
テーマ：[投稿のメインコピーを入力]
スタイル：シンプルミニマル、余白多め、ゴシック体
カラー：背景#FAFAF8、テキスト#1A1A1A、アクセント#3D6B4F
右下に「@tadanosuke.divizero」を小さく入れてください
```

## 7. 初期10本プラン（受け皿合格ラインへの最短ルート）

投稿0本 → 受け皿として機能する状態への最短順序。

```
1本目：タイプB（始めた理由）← 人間を最初に見せる
2本目：タイプD（発信してるのに売れない）← 共感で引き込む
3本目：タイプA（熱量が冷める話）← 気づきを渡す
4本目：タイプC（ファーストビューの役割）← ノウハウで信頼
5本目：タイプA（比較される前に完結させる）← 競合追走論点
6本目：タイプB（失敗した話 or 違うなと思った瞬間）← 人間を深掘り
7本目：タイプD（LP作ったのに問い合わせが来ない）← 共感
8本目：タイプC（LINE登録ページで外してはいけない1点）← ノウハウ
9本目：タイプA（ページは24時間の営業マン）← 気づき
10本目：タイプE（事業開示）← 信頼残高が溜まってから出す
```

10本が揃ったら営業開始可能。タイプEは必ず最後。

## 8. 発展：投稿ネタの補充ルール

ネタ切れを防ぐ仕組み。以下の3つのソースから常に補充する。

**ソース①：DMや会話から拾う**
相談者から「そうなんですよ、実は〜」と言われた内容はそのままタイプA・Dのネタになる。
個人が特定されない形に変えて使う。これが最も刺さる投稿になる。

**ソース②：ターゲット層の投稿から拾う**
まみや・セメントくん等のアフィリ・教育系アカウントが「悩み」を投稿していたら、
その悩みをタイプAかDに昇華する（引用ではなく、テーマとして使う）。

**ソース③：OS③の案件検証から拾う**
失注・受注の事後検証で出てきた「相手の言葉・課題・反応」をネタ化する。
これができれば、営業データが投稿コンテンツに自動変換される。
