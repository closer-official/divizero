# OS① 接触スクリーニングOS Ver4.0
（旧：接触可否・初回接触戦略分析OS Ver3.0）

---

## Ver4.0での設計転換

旧版は「接触する価値があるか」を精密に事前判定するOSだった。

新版は違う。

**除外条件に当たらない限り、原則すべて接触する。**

接触すべきだったかどうかの判定は、接触して結果が出た後にOS③で行う。

理由：

- 事前判定の精度は過去データ量に依存する。データが少ない段階での精密採点は検証不能な予測にすぎない
- SKIPした相手は永遠に答え合わせができない。接触した相手はすべて学習データになる
- リプ・DMの失敗コストは「返信が来ない」程度であり、機会損失コストの方がはるかに大きい

このOSの所要時間は **1案件60秒以内** とする。
60秒を超える分析をしたくなったら、それは事後（OS③）でやるべき分析である。

---

## STEP1：除外フィルター（SKIP判定）

以下のいずれか1つでも該当したらSKIP。該当しなければ次へ進む。

| # | 除外条件 | 判定基準 |
|---|---|---|
| 1 | 同業者 | Web制作・LP制作・HP制作を自ら販売している（※市場学習目的の交流は別枠で可） |
| 2 | 営業拒否明記 | プロフィール・固定ポストに「営業DMお断り」等の明記 |
| 3 | 事業性なし | 収益化意図が見えない完全趣味アカウント |
| 4 | 活動停止 | 最終投稿が3ヶ月以上前 |
| 5 | リスク案件 | 炎上中・係争中・誹謗中傷の常習 |

**SKIPの場合も必ずログに残す**（後日、SKIP判定自体の妥当性を抜き打ち検証するため）。

---

## STEP2：トラック判定

SKIPでなければ、以下の2択。「接触しない」という選択肢はない。

### ファストトラック（FT）：DM直行

以下のいずれかの**課題シグナル**が直近30投稿以内にあれば、リプ段階を飛ばしてDM試行へ直行する。

- HP・LP・Webに関する困りごとを自分から発信している（「HPどうしよう」「LP作りたいけど」等）
- 問い合わせ不足・予約不足・集客の悩みを発信している
- 制作者・解決策を探す発言をしている
- 「誰かいい人いませんか」系の募集

**重要：課題を発信している相手は競合にも見えている。顕在層に関係構築営業を適用するのは、最も受注しやすい層を最も遅く処理する行為である。FTは24時間以内にDM実行。**

### 通常トラック（NT）：リプ接触から開始

課題シグナルが見えない相手はすべてここ。リプ往復から入り、以降の進行はOS②に従う。

| 記入日 | |
| アカウント名 / URL | |
| 業種 | |
| フォロワー数 | |
| 推定商品 / 推定単価 | |
| 課題シグナル | 有（内容を引用）/ 無 |
| トラック判定 | SKIP / FT / NT |
| **事前仮説（1行・必須）** | 「この人は◯◯に困っている可能性。◯◯の話題なら反応するはず」 |

**事前仮説はこのOSで最も重要な出力である。**
仮説を書かずに接触すると、結果が出ても何も検証できず、案件が学習データにならない。
仮説の当たり外れはOS③で必ず答え合わせする。

## STEP4：初回接触案の生成

トラックに応じて生成する。3案も要らない。**実行する1案＋予備1案のみ。**

### FTの場合：DM文 2案

条件：
- 課題シグナルの投稿に具体的に触れること（テンプレ感の排除）
- 売り込み禁止・提案禁止
- 「その件、少し気になったので」程度の自然な入り
- 80〜150文字

### NTの場合：リプ文 2案

条件：
- 80文字以内
- 質問は1個まで（初回は質問なし推奨）
- 相手の直近投稿の内容に具体的に反応すること

## 旧版から廃止した項目と理由

| 廃止項目 | 廃止理由 |
|---|---|
| 契約可能性 0-100 | データなしでの採点は検証不能。事後にOS③で実結果と照合する方式に変更 |
| 課題顕在度 0-100 | 「課題シグナル有/無」の2値で十分。シグナル有＝FT直行 |
| 営業警戒度・世界観防衛度・DM許容度等の多軸スコア | 接触前に外から測れるものではない。OS②の実反応観測に移管 |
| 営業適性 S〜D | 接触可否の判断材料に使われ母数を削るため廃止。適性は事後に判明する |
| 推奨接触レベル Level0〜5 | FT/NTの2トラックに簡素化。段階の細分化は試行速度を落とす |
| 推定関心領域5個・課題候補3個 | 事前仮説1行に統合。多すぎる仮説は検証されない |
| GO/KEEP/HOLD/SKIP の4値判定 | KEEP・HOLDは「動かない理由」の温床。SKIP以外は全件接触に変更 |

## このOSの成功指標

- 1日あたりのスクリーニング処理件数（多いほど良い）
- 接触実行率＝（FT＋NT）÷ 全スクリーニング数（SKIPが3割を超えたら除外基準が厳しすぎる疑い）
- 事前仮説の的中率（OS③で集計。的中率が分かって初めて、事前判定の精密化に意味が出る）

---

## 移行ルール（試行期 → 最適化期）

- **試行期（クローズ案件50件まで、または3ヶ月）**：本OSの通り、除外以外は全件接触
- **最適化期（50件以降）**：OS③の集計データに基づき、「接触しても無駄だった層」の条件を除外フィルターに追加していく。事前判定の精密化はこの段階で初めて行う
